# API Documentation

OpenAPI is generated by SpringDoc at runtime:

- Swagger UI: `/swagger-ui/index.html`
- OpenAPI JSON: `/v3/api-docs`

## Implemented Auth API

| Method | Path | Roles | Description |
| --- | --- | --- | --- |
| `POST` | `/api/v1/auth/register` | Public | Create a user in a school tenant |
| `POST` | `/api/v1/auth/login` | Public | Authenticate and return JWT |
| `GET` | `/api/v1/auth/me` | Authenticated | Return current JWT principal |
| `GET` | `/api/v1/auth/health` | Public | Service health |

## Planned REST Resources

All collection APIs use pagination with `page`, `size`, and `sort`, and filtering with explicit query parameters.

```text
GET    /api/v1/students?page=0&size=20&classId=&sectionId=&status=
POST   /api/v1/students
GET    /api/v1/students/{id}
PUT    /api/v1/students/{id}
DELETE /api/v1/students/{id}

GET    /api/v1/teachers?page=0&size=20&subjectId=
POST   /api/v1/teachers
PUT    /api/v1/teachers/{id}/subjects

POST   /api/v1/attendance
GET    /api/v1/attendance?date=&classId=&sectionId=&studentId=

POST   /api/v1/fees/structures
POST   /api/v1/fees/payments
GET    /api/v1/fees/students/{studentId}/ledger

POST   /api/v1/exams
POST   /api/v1/exams/{examId}/results
GET    /api/v1/reports/report-cards/{studentId}

POST   /api/v1/notifications
GET    /api/v1/notifications/inbox
```
